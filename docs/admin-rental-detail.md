# 管理员租赁详情与排障

实现日期：2026-06-10

## 背景

租赁是用户订单、Sub2API 交付、OpenAI/Codex 本地反代、用量计费和 API Key 管理之间的核心连接点。此前管理员可以查看租赁列表、订单详情和全局反代请求列表，但当运维人员已经掌握租赁 ID、endpoint 或 Key 前缀时，仍需要跨多个页面拼接证据。

## 已实现能力

- 新增管理员接口 `GET /api/admin/rentals/:id`。
- 接口返回租赁本体、用户、商品、订单、限制、最近 API Key、最近用量记录和最近本地反代请求日志。
- 接口返回 `usageSummary`，聚合该租赁的用量数量、输入/输出 tokens、API 等价成本、买家计费和供给方收入。
- 接口返回 `proxyRequestSummary`，聚合该租赁关联的本地反代请求数量。
- 管理后台租赁列表新增 `Detail` 操作。
- 租赁详情面板展示交付信息、Sub2 user/key、endpoint、到期时间、限制额度、API Key、最近用量、最近反代请求和关联订单。
- 租赁状态、额度、API Key 状态、Key 轮换和过期批处理后，若当前租赁详情已打开，会自动刷新详情。

## 运维价值

- 管理员可以直接从租赁维度核对“是否已交付 endpoint/key”。
- 管理员可以直接判断该租赁是否产生了本地 OpenAI/Codex 反代请求。
- 管理员可以对比反代请求与用量账单，确认请求是否进入计费链路。
- 管理员可以在售后排查时围绕单个租赁闭环查看订单、Key、用量和请求证据。
