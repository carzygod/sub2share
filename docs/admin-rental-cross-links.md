# 管理员租赁交付横向钻取

实现日期：2026-06-12

## 背景

租赁通道是售出订单、API Key、本地 OpenAI/Codex 反代、Sub2 Key、用量和结算之间的交付枢纽。此前管理后台已经有租赁列表和租赁详情，但管理员从单个租赁继续定位用户、订单、商品、API Key、用量、反代请求或结算时，仍需要复制 ID 后切换页面搜索。

## 已实现范围

- `租赁通道` 列表行新增用户、订单、商品、API Key、用量和反代请求直达操作。
- `租赁详情` 顶部新增用户、订单、商品、用量和反代请求直达操作。
- `租赁详情` 的 API Key 表新增操作列，可打开 API Key 管理。
- `租赁详情` 的最近用量新增操作列，可打开用量、反代请求和结算。
- `租赁详情` 的最近反代请求新增操作列，可打开反代请求、API Key 和用量记录。

## 管理价值

- 管理员可以从单个租赁交付串联用户、售出订单、商品、API Key、用量、反代请求和结算。
- OpenAI/Codex 交付不可用、Key 停用、限额异常、用量争议和反代失败的定位路径更短。
- 复用既有管理列表、详情加载和权限边界，不新增绕过后台入口的隐藏操作。
