# 管理员供给方管理

实现日期：2026-06-10

## 背景

共享资源管理此前主要围绕 `SupplierResource` 展开，管理员可以创建资源、测试资源、查看资源详情和维护资源配置。但 `Supplier` 本体的状态、显示名和默认分成缺少独立后台入口，运营人员需要从资源或用户详情间接判断供给方情况。

## 已实现范围

- 新增管理员接口 `GET /api/admin/suppliers`。
- 新增管理员接口 `PATCH /api/admin/suppliers/:id`。
- 供给方列表支持分页、关键字搜索和状态筛选。
- 供给方列表返回用户、最近资源、资源数量和提现数量。
- 管理后台新增 `供给方` 导航入口。
- 管理后台供给方列表展示邮箱、显示名、状态、默认分成、资源/提现数量和最近资源。
- 管理员可维护供给方显示名、状态和默认分成。
- 供给方更新写入审计日志：`admin.supplier.update`。
- 供给方 CSV 导出支持邮箱、状态、默认分成、资源数、提现数和更新时间。

## 运营价值

- 管理员可以直接管理所有供给方，不必先进入某个共享资源。
- 供给方状态可以独立维护，便于处理暂停合作、禁用异常供给方和重新启用供给方。
- 默认分成可在供给方层面维护，为后续新资源创建或结算策略提供统一基础。

## 注意事项

供给方默认分成不会自动改写已创建共享资源的 `shareRate`。已经创建的资源仍通过共享资源配置入口独立维护。
