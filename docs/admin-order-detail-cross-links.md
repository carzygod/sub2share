# 管理员订单详情横向钻取

实现日期：2026-06-12

## 背景

售出订单详情已经聚合展示交付核查、余额流水、反代请求、订单项、租赁交付、租赁限制和 API Key。此前这些证据主要用于阅读，管理员从订单售后进入用户、余额、反代请求或租赁管理时，仍需要复制 ID 后切换页面搜索。

## 已实现范围

- `订单详情` 顶部新增 `打开用户` 操作。
- `钱包流水` 新增操作列，可按流水 ID 打开余额流水列表。
- `最近反代请求` 新增操作列，可按 `requestId` 打开反代请求列表。
- `订单项` 新增操作列，可按商品 ID 打开商品配置列表。
- `租赁交付` 新增操作列，可打开租赁通道并进入租赁详情。
- `租赁限制` 新增操作列，可打开租赁通道并进入租赁详情。
- `API Key` 新增操作列，可按 Key ID 打开 API Key 管理列表。
- 该能力同时覆盖 `售出情况` 与 `订单管理` 两个入口，因为它们共用订单详情组件。

## 管理价值

- 管理员从售出订单即可继续追踪用户情况、余额流水、反代请求、租赁交付和 API Key。
- 订单交付失败、退款复核、反代失败和用量争议的定位路径更短。
- 复用既有列表筛选、详情加载和权限边界，不新增绕过后台入口的隐藏操作。
