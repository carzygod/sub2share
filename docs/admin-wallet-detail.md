# 管理员钱包详情

实现日期：2026-06-10

## 背景

管理后台此前有钱包列表和全局余额流水页，但管理员核对单个钱包时需要手工复制钱包 ID 或用户 ID 到流水页搜索。调账后也缺少一个集中视图来确认余额、用户关系和最近流水是否一致。

本次补齐钱包详情入口，让余额管理从列表盘点推进为单钱包核对闭环。

## 后端接口

- 新增接口：`GET /api/admin/wallets/:id`
- 权限要求：`operator` 或 `admin`
- 返回内容：
  - 钱包基础余额：可用余额、冻结余额、累计充值、累计消费。
  - 钱包所属用户和用户角色。
  - 最近 100 条余额流水。
  - 流水汇总：`transactionSummary._count` 和 `transactionSummary._sum.amount`。

## 管理员入口

管理后台 `余额管理` 页面增强：

- 钱包列表新增 `详情` 操作。
- 钱包详情展示钱包 ID、用户 ID、余额、冻结、累计充值、累计消费、流水数量和更新时间。
- 钱包详情展示最近 20 条余额流水，包含类型、金额、余额后、引用、备注和时间。
- 管理员调账后，如果当前钱包详情正在打开，会自动刷新详情。

## 验收方式

本地验证：

- `npm --prefix user/apps/api run typecheck`
- `npm --prefix user/apps/admin run typecheck`
- `npm --prefix user/apps/api run build`
- `npm --prefix user/apps/admin run build`

功能验证建议：

1. 进入后台 `余额管理`。
2. 点击任一钱包的 `详情`。
3. 确认详情中展示该钱包的最近余额流水。
4. 对该用户执行一次管理员调账。
5. 确认钱包列表和打开的钱包详情刷新后余额一致，最近流水出现 `adjustment` 记录。
