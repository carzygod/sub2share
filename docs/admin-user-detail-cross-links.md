# 管理员用户详情横向钻取

实现日期：2026-06-12

## 背景

管理员目标不只是看到单个用户资料，还需要从用户视角快速进入余额、售出、租赁、API Key、共享资源和提现等运营对象。此前用户详情页已经展示这些关联数据，但行内对象主要是只读文本，管理员需要复制 ID 后再切换列表搜索。

## 已实现范围

- `用户详情` 顶部新增 `打开余额` 操作，可直接打开该用户钱包筛选结果。
- `最近钱包流水` 新增操作列，可按流水 ID 打开余额流水列表。
- `最近订单` 新增操作列，可打开订单管理并进入订单详情。
- `最近租赁` 新增操作列，可打开租赁通道并进入租赁详情。
- `API Key` 新增操作列，可打开 API Key 管理并按 Key ID 筛选。
- `供给资源` 新增操作列，可打开共享资源列表并进入资源详情。
- `提现记录` 新增操作列，可打开提现管理并按提现 ID 筛选。

## 管理价值

- 管理员从一个用户详情页即可串联用户情况、余额情况、售出情况和共享情况。
- 处理售后、余额争议、资源供给问题时，不再依赖手工复制 ID 和切换搜索。
- 复用既有列表筛选、详情打开和权限控制，不新增绕过管理入口的隐藏操作。
