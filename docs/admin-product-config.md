# 管理员商品配置维护

实现日期：2026-06-10

## 背景

后台 API 已支持 `PATCH /api/admin/products/:id`，但管理后台此前只能创建商品和通过快捷按钮调整商品状态，不能直接维护商品名称、描述、资源类型、计费模式和状态。对于售卖入口运营来说，商品基础配置与价格档位一样需要在后台闭环管理。

## 已实现范围

- 管理后台商品列表为每个商品新增内联配置表单。
- 管理员可修改：
  - 商品名称
  - 商品描述
  - 资源类型
  - 计费模式
  - 商品状态
- 保存商品前会弹出确认，展示关键配置值。
- 保存后刷新商品列表，确保新配置立即可见。
- 后端 `PATCH /api/admin/products/:id` 支持 `description=null`，用于清空商品描述。
- 空更新会被拒绝，避免写入无意义审计。

## 运营价值

- 管理员可以在后台完整维护售卖商品，不再只能创建和上下架。
- 商品基础信息、价格档位、套餐限额和售出交付形成更完整的运营闭环。
- 描述支持清空，便于移除过期说明或错误营销文案。
