# 管理员 API Key 横向钻取

实现日期：2026-06-12

## 背景

API Key 是买家访问本地 OpenAI/Codex 反代的入口凭据，也是租赁交付、用户身份、订单、商品、反代请求和用量记录之间的关键连接点。此前 `API Key 管理` 列表已经支持筛选、启用、停用和批量状态调整，但管理员从单个 Key 继续定位用户、订单、租赁、商品、反代请求或用量时，仍需要复制 ID 后切换页面搜索。

## 已实现范围

- `API Key 管理` 列表行新增横向钻取操作。
- 每个 Key 可按已有关联 ID 打开：
  - 用户管理。
  - 订单管理并进入订单详情。
  - 租赁通道并进入租赁详情。
  - 商品配置。
  - OpenAI/Codex 反代请求列表。
  - 用量记录列表。
- 保留既有启用、停用和批量状态操作。

## 管理价值

- 管理员可以从单个 API Key 串联用户、售出订单、租赁交付、商品、反代请求和用量证据。
- Key 停用、泄露处置、客户请求失败、限额争议和 `/v1/responses` 排障路径更短。
- 复用既有列表筛选、详情加载和权限边界，不新增绕过后台入口的隐藏操作。
