# 管理后台高风险操作确认

实现日期：2026-06-10

## 背景

管理后台已经覆盖用户、余额、售出、租赁、共享资源、结算、提现和 Sub2 反代运维。此前多数高风险动作可以一键触发，虽然后端已有权限、状态机和审计日志，但前端缺少误触确认。

本次补齐轻量二次确认层，降低管理员在日常运营中误点退款、封禁、停用 Key、关闭租赁或打款的风险。

## 已覆盖动作

以下操作在执行前会要求管理员确认，部分操作还会要求填写审计备注：

- 用户禁用、封禁和角色调整。
- 管理员手动调整余额。
- 商品或价格档位下线。
- 租赁暂停、关闭、限额调整、批量过期处理和 Key 轮换。
- API Key 停用。
- 订单取消和退款，并写入管理员输入的备注。
- 共享资源暂停、异常、禁用和配置调整。
- 提现审批、驳回、取消、打款；打款仍要求填写 payout reference。
- 到期结算释放。
- 系统维护。
- Sub2 绑定修复。
- OpenAI refresh token 应用。

## 设计边界

- 当前实现是前端确认层，用于减少误触；后端权限、状态机、审计日志仍是最终安全边界。
- `operator` / `admin` 的权限边界不变。
- 订单、提现等支持备注的后端接口会继续沉淀管理员输入的备注。
- 更严格的二次验证、管理员密码复核或 2FA 可作为后续增强。

## 验收方式

本地验证：

- `npm --prefix user/apps/admin run typecheck`
- `npm --prefix user/apps/admin run build`

功能验证建议：

1. 进入管理后台。
2. 点击退款、禁用用户、关闭租赁、停用 API Key 或提现打款等按钮。
3. 确认浏览器会弹出确认或备注输入框。
4. 取消弹窗后确认不会发出对应请求。
5. 确认执行后，相关后端接口仍写入原有审计日志。
