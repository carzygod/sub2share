# 管理员价格档位配置维护

实现日期：2026-06-10

## 背景

后台 API 已支持 `PATCH /api/admin/product-prices/:id`，但管理后台页面此前只能创建价格档位和启停价格，不能直接修改已有价格的租期、并发、RPM/TPM、请求数、消费上限、折扣率和倍率。对于运营人员来说，这会导致套餐配置调整仍需要调用接口或直接改库。

## 已实现范围

- 管理后台每个价格档位新增内联配置表单。
- 管理员可修改价格名称、固定价格、租期、并发、RPM、TPM、请求数、消费上限、折扣率、倍率和状态。
- 保存价格前会弹出高风险操作确认，并展示关键变更字段。
- 保存后刷新商品列表，确保新配置立即在后台可见。
- 后端价格更新 schema 支持对可选字段传入 `null`，用于清空：
  - `durationDays`
  - `rpmLimit`
  - `tpmLimit`
  - `requestLimit`
  - `spendLimit`
- `admin.product_price.update` 审计记录补充租期、并发、RPM/TPM、请求数、消费上限、折扣率和倍率。

## 运营价值

- 管理员不再只能创建新档位或启停旧档位，可以直接维护已有套餐配置。
- 售卖入口的价格、限额和计费倍率可在后台闭环维护。
- 可选限额支持清空，便于将套餐从有限额调整为不限额。

## 注意事项

价格档位配置只影响后续新下单租赁。已经售出的租赁仍以其 `RentalLimit` 为准，需要通过租赁管理入口单独调整。
