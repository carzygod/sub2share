# 管理员余额与流水横向钻取

实现日期：2026-06-12

## 背景

余额账户和余额流水是充值、退款、订单扣费、用量扣费与提现冻结的核心证据。此前管理后台已经有余额账户列表、余额详情和全局余额流水列表，但这些页面主要用于读取和筛选。管理员从一条流水继续定位用户、订单、用量或提现时，仍需要复制 `refId` 后切换页面搜索。

## 已实现范围

- `余额管理` 列表行新增 `用户` 与 `流水` 操作。
- `余额详情` 顶部新增 `打开用户` 与 `打开流水` 操作。
- `余额详情` 的最近余额流水新增 `操作` 列。
- 全局 `余额流水` 列表新增 `操作` 列。
- 流水行内操作覆盖：
  - 按钱包 ID 打开余额账户列表。
  - 按用户 ID 打开用户管理并进入用户详情。
  - 按流水 ID 打开余额流水列表。
  - `refType=order` 时打开订单管理并进入订单详情。
  - `refType=usage` 时打开用量记录列表。
  - `refType=withdrawal` 时打开提现管理列表。

## 管理价值

- 管理员可以从余额账户或单条流水直接串联用户情况、售出订单、反代用量和提现记录。
- 充值争议、退款复核、用量扣费异常和提现冻结排查路径更短。
- 复用既有管理列表、详情加载和权限边界，不新增绕过后台入口的隐藏操作。
