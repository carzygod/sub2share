# 管理员用户角色管理

## 背景

管理员后台已经支持创建用户时指定角色，也能查看用户详情，但缺少创建后的角色调整入口。对于运营系统而言，`buyer`、`supplier`、`operator`、`admin` 角色决定了用户能否进入后台、管理资源、处理余额和操作高风险接口，因此角色管理属于用户管理闭环的一部分。

## 已实现

新增接口：

```text
PATCH /api/admin/users/:id/roles
```

权限：

- `admin`

请求体：

```json
{
  "roles": ["buyer", "supplier"]
}
```

支持角色：

- `buyer`
- `supplier`
- `operator`
- `admin`

处理行为：

- 对提交角色去重。
- 删除目标用户已不在提交集合中的旧角色。
- 补齐提交集合中的新角色。
- 返回更新后的用户、角色、钱包、供给方和计数信息。
- 写入审计日志 `admin.user.roles`。

## 安全保护

- 管理员不能移除自己的 `admin` 角色。
- 当系统仅剩一个 `admin` 角色时，不能移除该角色。
- 接口仍经过后台管理员鉴权，`operator` 只能查看，不能修改角色。

## 后台入口

用户详情面板新增角色编辑表单，格式与创建用户一致，使用英文角色名并用逗号分隔：

```text
buyer, supplier, operator
```

提交成功后刷新用户列表与当前详情。

## 可用性结论

该能力补齐了管理员对用户权限的生命周期管理。管理员现在不仅能创建用户、调整状态和余额，也能在用户运营过程中收敛后台权限、供给方权限和管理员权限。
