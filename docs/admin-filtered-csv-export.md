# 管理后台筛选结果 CSV 导出

实现日期：2026-06-10

## 背景

管理员需要按用户、余额、售出、租赁、反代请求等维度做运营盘点。此前后台 CSV 只导出当前页，遇到多页结果时需要手动翻页并多次合并，容易遗漏数据。

本次改造把列表导出升级为“导出全部筛选”：保留当前筛选条件，前端按分页接口逐页拉取数据后生成一份 CSV。

## 覆盖范围

以下管理列表支持按当前筛选条件导出全部分页数据：

- 用户管理
- 余额管理
- 余额流水
- 用量记录
- 商品管理
- 订单管理
- 租赁通道
- 反代请求
- 共享资源
- 结算管理
- 提现管理
- 操作审计

售出情况页仍保留独立的售出订单导出，它使用 `/api/admin/sales` 返回的售出订单快照。

## 实现方式

- 复用现有分页接口，不新增后端导出接口。
- 导出时强制使用 `pageSize=200`，这是当前后台列表接口允许的最大分页大小。
- 从第 1 页开始读取 `totalPages`，直到当前筛选条件下全部页面拉取完成。
- 文件名使用 `filtered-all` 后缀，便于和早期 `current-page` 导出区分。
- 导出过程中后台顶部消息会显示当前已拉取数量和总数。
- 若已有导出任务在进行，新的导出点击会提示当前任务尚未结束。

## 验收方式

本地验证：

- `npm --prefix user/apps/admin run typecheck`
- `npm --prefix user/apps/admin run build`

线上验证建议：

1. 进入任一列表页，例如 `订单管理`。
2. 设置搜索或状态筛选。
3. 点击 `导出全部筛选`。
4. 确认 CSV 行数等于当前筛选后的列表 `total`。
5. 对 `反代请求` 页面使用状态码或错误码筛选后重复导出，确认导出的都是筛选后的请求记录。
