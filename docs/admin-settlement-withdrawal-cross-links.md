# 管理员结算提现横向钻取

实现日期：2026-06-12

## 背景

结算记录和提现记录是共享资源收益链路的尾端，连接供给方、共享资源、用量、售出扣费和实际打款。此前 `供给方结算` 与 `提现管理` 已经支持筛选、释放到期结算、录入提现和状态流转，但管理员从单条结算或提现继续定位用户、共享资源、用量或对应结算时，仍需要复制 ID 后切换页面搜索。

## 已实现范围

- `供给方结算` 列表新增 `操作` 列。
- 每条结算可按已有关联 ID 打开：
  - 供给方用户。
  - 共享资源详情。
  - 用量记录列表。
  - 提现管理列表。
- `提现管理` 列表新增横向钻取操作。
- 每条提现可按已有关联 ID 打开：
  - 供给方用户。
  - 首个分配结算关联的共享资源。
  - 首个分配结算关联的用量记录。
  - 首个分配结算记录。
- `GET /api/admin/withdrawals` 搜索新增支持提现分配 ID 和结算记录 ID。
- 结算和提现空列表补充占位行，避免只有表头时误判页面未加载。

## 管理价值

- 管理员可以从结算或提现串联供给方、共享资源、用量、结算和打款记录。
- 供应商收益争议、提现审核、打款复核和共享资源分润排障路径更短。
- 复用既有列表筛选、详情加载和权限边界，不新增绕过后台入口的隐藏操作。
